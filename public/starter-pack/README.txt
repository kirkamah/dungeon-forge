СТАРТОВЫЙ НАБОР ТАЙЛОВ (Dungeon Forge)
=======================================

Сюда кладётся готовый набор тайлов, который автоматически появляется
у пользователя при первом запуске приложения (и его можно перезагрузить
кнопкой «Загрузить стартовый набор» в Настройках).

КАК ДОБАВИТЬ СВОИ ТАЙЛЫ
-----------------------
1. Положи квадратные PNG/JPG картинки тайлов в эту папку (public/starter-pack/).
2. Пропиши каждый файл в manifest.json в массиве "tiles".
3. Пересобери приложение (npm run build / npm run dist) — картинки из public/
   попадут в сборку автоматически.

ФОРМАТ manifest.json
--------------------
{
  "name": "Подземелье",                  // название набора
  "description": "Базовые каменные тайлы",
  "tiles": [
    { "file": "straight.png",  "type": "STRAIGHT",   "name": "Коридор" },
    { "file": "corner.png",    "type": "CORNER",     "name": "Угол" },
    { "file": "t.png",         "type": "T_JUNCTION", "name": "Т-перекрёсток" },
    { "file": "cross.png",     "type": "CROSSROADS", "name": "Перекрёсток" },
    { "file": "deadend.png",   "type": "DEAD_END",   "name": "Тупик" },
    { "file": "room.png",      "type": "ROOM_SMALL", "name": "Комната", "openings": 15 },
    { "file": "entrance.png",  "type": "ENTRANCE",   "name": "Вход" },
    { "file": "exit.png",      "type": "EXIT",       "name": "Выход" }
  ]
}

ПОЛЯ ТАЙЛА
----------
file     — имя файла картинки в этой папке (обязательно).
type     — тип тайла. Допустимые значения:
           STRAIGHT, CORNER, T_JUNCTION, CROSSROADS, DEAD_END,
           ROOM_SMALL, ROOM_LARGE, ENTRANCE, EXIT
name     — отображаемое имя (необязательно).
openings — маска проходов (необязательно; нужна в основном для комнат).
           Биты сторон: Север=1, Восток=2, Юг=4, Запад=8 — сложи нужные.
           Примеры: коридор С+Ю = 1+4 = 5; все стороны = 15.
           Если не указать, берётся стандартная маска для типа.

Картинки должны быть квадратными. Несколько файлов одного типа = разные
варианты (выбираются случайно при генерации).
